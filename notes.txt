while deploying it on render free tier which has a limit of 512 mb, i encountered 3 problems:
Problem 1 — Out of memory (512MB limit exceeded)
Problem 2 — law-ai/InLegalBERT not loading as a SentenceTransformer
Problem 3 — Port not detected (FastAPI not binding in time)

FIXES
Step 1 — Fix requirements.txt to use the CPU-only slim torch
The biggest memory killer is torch itself. The CPU build you are downloading (torch-2.4.0+cpu) is 195MB compressed but expands to ~700MB in memory at runtime. On Render's free 512MB tier, loading torch + BGE-Base + InLegalBERT simultaneously exceeds the limit immediately.
Replace your requirements.txt with this slimmed version
The key change is --extra-index-url https://download.pytorch.org/whl/cpu at the top. Without this, pip downloads the full CUDA-enabled torch (~2GB) even on a CPU server. With it, pip fetches the CPU-only build (~200MB compressed, ~450MB expanded).
